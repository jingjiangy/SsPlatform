## 1. 项目概述

本平台用于管理世界模型素材、执行机器人任务评测、维护设备与配件信息。核心业务流程：

```
素材库（世界模型视频）→ 评测任务 → 评测记录（视频录制 + 打分）→ 统计分析
```

主要功能模块：

| 模块 | 说明 |
|------|------|
| 系统管理 | 角色管理、账号管理 |
| 世界模型素材库 | 素材版本管理、评测模板 |
| 评测任务 | 任务创建、评测录入、步骤打分、统计 |
| 设备管理 | 设备型号、机器人、配件、故障记录 |
| 接口文档 | 在线 API 文档（Swagger） |

## 4. 快速启动

### 环境要求

- Python 3.10+
- Node.js 18+
- MongoDB（本地默认 `mongodb://localhost:27017`）

### 一键启动（推荐）

```bash
cd /path/to/SsPlatform
chmod +x start-dev.sh
./start-dev.sh
```

脚本会自动：
1. 检测本机 LAN IP
2. 生成 HTTPS 自签证书（若不存在）
3. 启动后端（端口 8000）
4. 启动前端（端口 5173）

支持环境变量覆盖端口：

```bash
BACK_PORT=9000 FRONT_PORT=5174 ./start-dev.sh
```

### 手动启动

```bash
# 后端
cd backend
pip install -r ../requirements.txt
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 前端（新终端）
cd frontend
npm install
npm run dev -- --host 0.0.0.0 --port 5173
```

### 默认账号

| 用户名 | 密码 | 角色 |
|--------|------|------|
| admin | admin123 | 管理员 |

首次启动时自动创建，无需手动初始化数据库。

## 6. 模块说明

### 6.1 账号管理

字段：用户 ID、用户名、密码（加密存储）、角色、手机号、创建时间、更新时间

操作：添加、编辑、删除用户；支持重置密码

### 6.2 角色管理

字段：角色 ID、角色名称、角色 Code、模块权限列表、创建时间

操作：添加、编辑、删除角色；权限按模块勾选

### 6.3 素材库

**父级素材字段**：素材 ID、素材名称、描述、类型、状态、版本号、视频封面、创建/更新时间、创建/更新人

**素材类型**：原子任务、长程任务、PR 任务、试采集任务

**素材状态**：搁置、进行中、测试中、丢弃、PR 已布、可备用

**版本号规则**：一位小数，手动输入（1.0 → 1.1 → ... → 1.9 → 2.0）

**子版本**：每个父级素材下可创建多个子版本，点击父级名称展开子版本列表

**视频上传**：格式 mp4/webm，大小不超过 1000MB

**评测详情入口**：从素材列表点击「评测详情」，携带素材 ID 跳转至评测任务页

### 6.4 评测模板

字段：模板 ID、模板名称、描述、状态、版本号、步骤列表（步骤名称 + 满分值）、创建/更新时间

操作：创建、编辑、删除模板；步骤支持动态增删

### 6.5 评测任务

**任务字段**：任务 ID、描述、类型、状态、版本号、关联素材、关联评测模板、步骤总分、评测次数（成功/总数）、成功率、平均视频时长、创建/更新时间

**任务类型**：实验任务、PR 任务

**任务状态**：待评测、进行中、已完成

**自动计算字段**：
- 评测次数：成功次数 / 总次数
- 成功率：成功次数 ÷ 总次数 × 100%，默认 0%
- 平均视频时长：有视频的记录总时长 ÷ 有视频的记录数，默认 0s
- 步骤平均分：每个步骤得分的均值

**评测录入（评测详情页）**：

| 字段 | 说明 |
|------|------|
| 动作执行描述 | 长文本，描述本次执行内容 |
| 视频录制 | 调用摄像头录制；支持开始录制、保存录制、放弃录制；保存后自动上传并显示封面 |
| 视频上传 | 也可直接上传本地视频文件 |
| 执行结果 | 下拉：成功 / 失败 |
| 执行时长 | 自动从视频时长读取（秒），默认 0 |
| 步骤打分 | 若任务关联了评测模板，右侧显示步骤列表，每步可单独打分，自动汇总总分 |

**提交逻辑**：若视频正在录制中，点击提交时自动先保存视频再提交记录

### 6.6 设备型号

字段：型号 ID、型号名称、描述、状态、创建/更新时间

操作：添加、编辑、删除

### 6.7 机器人管理

字段：机器人 ID、名称、关联设备型号、状态（在线/离线/故障）、版本号、创建/更新时间

操作：添加、编辑、删除

### 6.8 配件管理

字段：配件 ID、名称、描述、备注、状态（在线/离线/故障）、数量、关联机器人（可选）、图片、创建/更新时间、创建/更新人

**列表筛选**：按机器人筛选（全部 / 未关联 / 指定机器人）

**历史记录**：
- 出入库记录：时间、执行人、出入库类型、数量变化
- 更新记录：时间、执行人、更新后数量、更新后状态、更新后备注

每次保存配件信息自动生成一条更新流水；备注字段与历史记录中的「更新后备注」同步

**图片上传**：格式 jpg/png/jpeg，大小不超过 100MB

### 6.9 故障记录

字段：故障 ID、名称、描述、状态（待处理/处理中/已关闭）、关联机器人、维修人、故障图片、创建/更新时间

**列表筛选**：按机器人筛选（全部 / 未关联 / 指定机器人）

操作：添加、编辑、删除；支持上传故障图片

## 8. API 接口

后端启动后访问 Swagger 文档：`http://localhost:8000/docs`

### 认证

所有接口（除登录外）需在请求头携带：

```
Authorization: Bearer <access_token>
```

### 主要端点

| 模块 | 前缀 | 主要端点 |
|------|------|---------|
| 认证 | `/api/auth` | `POST /login`、`GET /me` |
| 用户 | `/api/users` | CRUD |
| 角色 | `/api/roles` | CRUD |
| 素材 | `/api/materials` | CRUD、`POST /{id}/upload-video`、`GET /device-model-options` |
| 评测 | `/api/evaluations` | 任务 CRUD、记录 CRUD、`POST /tasks/{id}/records/upload-video`、模板 CRUD |
| 机器人 | `/api/robots` | CRUD |
| 设备型号 | `/api/device-models` | CRUD |
| 配件 | `/api/parts` | CRUD、出入库记录 |
| 故障记录 | `/api/fault-records` | CRUD |
| API 配置 | `/api/apis` | CRUD |

### 文件上传

- 视频：`multipart/form-data`，字段名 `file`，返回 `{ "video_url": "/static/uploads/..." }`
- 图片：同上，返回 `{ "image_url": "/static/uploads/..." }`
- 静态文件访问：`http://localhost:8000/static/uploads/<filename>`

### 页面路由

| 路径 | 页面 | 说明 |
|------|------|------|
| `/login` | LoginView | 登录页 |
| `/users` | UsersView | 账号管理 |
| `/roles` | RolesView | 角色管理 |
| `/materials` | MaterialsView | 素材库 |
| `/materials/:id/versions` | MaterialVersionsView | 子版本列表 |
| `/eval-templates` | EvalTemplatesView | 评测模板 |
| `/evaluations` | EvaluationsView | 评测任务列表 |
| `/evaluations/tasks/:taskId` | EvalDetailView | 评测详情（录入页） |
| `/eval-by-material` | EvalByMaterialView | 按素材查看评测 |
| `/device-models` | DeviceModelsView | 设备型号 |
| `/robots` | RobotsView | 机器人管理 |
| `/parts` | PartsView | 配件管理 |
| `/fault-records` | FaultRecordsView | 故障记录 |
| `/api-docs` | ApiDocsView | 接口文档 |

### 认证流程

1. 登录成功后 Token 存入 `localStorage["auth-storage"]`（Zustand persist）
2. Axios 请求拦截器自动读取并注入 `Authorization: Bearer <token>`
3. 响应拦截器检测 401，自动清除 Token 并跳转登录页
4. 路由守卫根据 `isLoggedIn` 状态决定是否重定向

### 权限控制

- `canSeeModule(mod)` — 控制侧边栏菜单显示
- `canWrite*()` 系列函数 — 控制新增/编辑/删除按钮显示
- 权限数据来源：登录响应中的 `modules` 和 `perms` 字段，降级时按角色推断

## 10. 配置说明

### 后端环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `MONGODB_URL` | `mongodb://localhost:27017` | MongoDB 连接串 |
| `DB_NAME` | `model_eval` | 数据库名 |
| `SECRET_KEY` | （内置默认值） | JWT 签名密钥，生产环境务必修改 |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | `1440`（24h） | Token 有效期（分钟） |
| `MAX_VIDEO_SIZE_MB` | `1000` | 视频上传大小限制 |
| `MAX_IMAGE_SIZE_MB` | `100` | 图片上传大小限制 |

### 前端环境变量

在 `frontend/.env` 中配置：

```env
VITE_API_BASE=http://localhost:8000
```

不设置时默认为空字符串（同域代理）。

### 生产部署建议

1. 修改 `SECRET_KEY` 为随机强密钥
2. 配置 MongoDB 认证
3. 使用 Nginx 反向代理前后端，统一域名
4. 前端执行 `npm run build`，将 `dist/` 目录交由 Nginx 托管
5. 后端去掉 `--reload`，使用 `gunicorn` 或 `supervisor` 管理进程
