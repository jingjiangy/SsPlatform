# SsPlatform 开发环境配置

本文说明如何在本地运行 **FastAPI 后端** 与 **Vue 3 + Vite 前端**，并与 MongoDB 联调。

## 1. 必备环境

| 软件 | 说明 |
| --- | --- |
| **Conda**（推荐） | 用于管理 Python 环境；**默认按 Conda 说明配置**，命令见下一节 |
| **Python 3.10+** | 建议与团队统一小版本，激活环境后 `python -V` 检查 |
| **Node.js 18+** | 与 Vite 5 兼容；`node -v` / `npm -v` 检查 |
| **MongoDB** | 本机或远程实例；开发默认 `mongodb://localhost:27017` |
| **OpenSSL**（可选） | 仅用于生成本地 HTTPS 自签证书（`frontend/certs/`，见下文） |

## 2. 拉代码与安装依赖

在仓库根目录执行（**默认使用 Conda** 创建与激活环境）：

```bash
cd /path/to/SsPlatform

# 后端：Conda 环境（名称可自定，如 ssplatform）
conda create -n ssplatform python=3.13 -y
conda activate ssplatform
pip install -U pip
pip install -r requirements.txt

# 前端
cd frontend
npm install
cd ..
```

之后每次开发前先执行 `conda activate ssplatform`（与上面环境名一致）。若你更习惯 venv，也可用 `python3 -m venv .venv` + `source .venv/bin/activate`，与 Conda 二选一即可。

说明：根目录的 `requirements.txt` 为后端依赖；`backend` 下无独立 `requirements.txt`，启动目录为 `backend/`（包名为 `app`）。

## 3. 环境变量

### 3.1 后端（`backend/.env`）

以模板为准：

```bash
cp backend/.env.example backend/.env
```

常用项（与 `app/config.py` 对应）：

| 变量 | 说明 | 开发示例 |
| --- | --- | --- |
| `MONGODB_URI` | MongoDB 连接串 | `mongodb://localhost:27017` |
| `MONGODB_DB` | 数据库名 | `model_eval`（可与示例一致或自定） |
| `SECRET_KEY` | JWT 密钥 | 开发可随机长字符串，**勿提交真实密钥** |

可选（有默认值，一般可不写）：`ALGORITHM`、`ACCESS_TOKEN_EXPIRE_MINUTES`、`UPLOAD_DIR`、`MAX_VIDEO_MB`、`MAX_IMAGE_MB` 等。

### 3.2 前端（`VITE_API_BASE`）

- **本地用 Vite 开发（默认 `npm run dev`）**  
  `vite.config.ts` 已把 `/api`、`/static`、`/docs` 等代理到 `http://127.0.0.1:8000`，一般 **无需** 设置 `VITE_API_BASE`（`http.ts` 中为空字符串表示走当前站点 + 相对路径，与代理配合）。
- **需直连后端、或打手机 / 非代理场景**  
  在 `frontend` 下新建 `.env.local`（不提交）：

  ```env
  VITE_API_BASE=http://你的机器IP:8000
  ```

  以实际后端可访问地址为准，末尾不要多写多余路径。

## 4. 启动数据库

本机已安装 MongoDB 时，确保服务监听 `27017`（或修改 `MONGODB_URI`）。无 Mongo 时，可用 Docker 等方式自行拉起，与连接串一致即可。

## 5. 运行方式

### 5.1 推荐：一条命令同时起前后端

在仓库根目录（需已安装 `frontend` 依赖，**并已 `conda activate` 到装好后端依赖的环境**）：

```bash
./start-dev.sh
```

- 默认后端 `BACK_PORT=8000`，前端 `FRONT_PORT=5173`。
- 可覆盖：  
  `BACK_PORT=9000 FRONT_PORT=5174 ./start-dev.sh`
- 若存在 `frontend/certs/localhost-*.pem`，Vite 会以 **HTTPS** 起开发服；否则可运行 `cd frontend && npm run gen-cert` 生成自签证书（需 OpenSSL）。

脚本会在退出时结束子进程；**注意**其中包含对 `pkill` 的清理逻辑，请确认与当前本机多 Python 进程不冲突（团队机器上如需更温和的方式可改脚本）。

### 5.2 分终端手动启动

**终端 1：后端**（已 `conda activate` 到上述环境）

```bash
cd backend
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

**终端 2：前端**

```bash
cd frontend
npm run dev -- --host 0.0.0.0 --port 5173
```

- 本机访问：`http://localhost:5173` 或 `https://localhost:5173`（有证书时，浏览器会提示自签，开发环境可继续访问）。

## 6. 功能入口与联调

| 项目 | 地址 / 说明 |
| --- | --- |
| 前端登录页 | `http://localhost:5173/login`（或 https + 自签时对应端口） |
| 健康检查 | `GET http://localhost:8000/api/health` |
| API 文档（联调时通常走代理） | `http://localhost:5173/docs`（代理到后端的 FastAPI `/docs`） |
| 上传静态资源 | 后端将文件放在 `upload_dir`（默认 `uploads`），经 `/static/uploads/...` 提供 |

**首次无数据时**会自动种子角色与「第一次」默认管理员（空库时写入，以应用启动逻辑为准）：

- 用户名 / 密码：**`admin` / `admin123`**

**首次登录后请按安全策略修改密码。**

## 7. 端口与代理一览

- 前端开发服务器：`5173`（可改 `FRONT_PORT` 或 `npm run dev -- --port`）。
- 后端：`8000`（`BACK_PORT` 或 `uvicorn --port`）。
- Vite 代理：将前端的 `/api`、`/static`、`/docs`、`/redoc`、`/openapi.json` 转到 `http://127.0.0.1:8000`，便于同域/避免 CORS 问题。

## 8. 常见问题

1. **前端能开、但接口 502 / 连不上**  
   确认本机 8000 上 uvicorn 已起，且 MongoDB 可连，查看后端终端日志。

2. **Mongo 连接失败**  
   检查 `MONGODB_URI`、防火墙、Mongo 是否对当前用户鉴权等。

3. **手机访问本机 https 与相机等 API**  
   按 `start-dev.sh` 说明信任自签证书，必要时配置 `VITE_API_BASE` 为手机可达的后端地址。

4. **macOS 上对外置盘/下载目录写入 `Operation not permitted`**  
  在「系统设置 → 隐私与安全性」中为 **终端/IDE** 打开「可移动卷」、「下载文件夹」或「完全磁盘访问」等，并重开终端（属系统行为，与代码无关）。
